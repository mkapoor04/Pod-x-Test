§ 05 — STEAL THIS

Three tips for the toolkit.

Small things that pay for themselves on day one.

01 AEP · Query Service

Prototype before you promote.

Run exploratory queries in the ad-hoc workspace before turning them into a Data Distiller dataset. You'll catch joins-from-hell at the cheapest possible point.

02 AEP · XDM

Always namespace custom fields.

Drop everything custom under _tenant.{your_org} — never the root. Future-proofs you against schema collisions and keeps Adobe upgrades from rewriting your fields.

03 CJA · Calculated Metrics

Save once, reuse forever.

An expensive metric used in three dashboards is three times the cost. Save it as a component, share via library, swap the tax for a single source of truth.

§ 06 — THE DECODER

Glossary, expanded.

Hover or tap any panel to see the full thing. Five acronyms we say too often.

RTCDP
Real-Time Customer Data Platform

The Adobe SaaS layer that ingests, unifies, segments, and activates customer profiles in milliseconds — built on top of AEP. If you've ever pushed a segment to Facebook Ads in real time, this is what you used.

→ EXPANDED
XDM
Experience Data Model

Adobe's open standard for describing customer-experience data. Every field in AEP / AJO / CJA conforms to an XDM schema — that's how a clickstream event from one team can talk to a journey built by another.

→ EXPAND
IMS
Identity Management Service

Adobe's identity backbone. Stitches the same person across devices, channels, and login states using the Identity Graph. Without it, "unified profile" is just three rows in a table.

→ EXPAND
EDGE
Adobe Edge Network

The globally-distributed compute layer that lets AJO score and decide on a profile in <100ms. If your journey "feels real-time" to a customer, the Edge Network is the reason.

→ EXPAND
B2P
Business-to-Person

The unified view Adobe is pushing for orgs that have both B2B and B2C motions — model the human, not the channel. Reduces duplicate profiles when the same person shows up as both a buyer and a consumer.

→ EXPAND
